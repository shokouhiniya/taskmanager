## ⚙️ تنظیمات لازم برای دسترسی آنلاین

اگر سایت هنوز deploy نشده یا دسترسی ندارید، این مراحل رو انجام بدید:

### 1️⃣ بررسی وضعیت سرور

```bash

ssh user@taskmanager.mardomi.org

# چک کردن سرویس‌ها

# یا

```

### 2️⃣ بررسی Backend

```bash
# چک کردن Backend
curl http://localhost:3000

# اگر خطا داد، restart کنید
docker restart taskmanager_backend
# یا
pm2 restart taskmanager-backend
```

### 3️⃣ بررسی Frontend

```bash
# چک کردن Frontend
curl http://localhost:4173

# اگر خطا داد، restart کنید
docker restart taskmanager_frontend
# یا
pm2 restart taskmanager-frontend
```

### 4️⃣ بررسی Nginx

```bash
# چک کردن تنظیمات
sudo nginx -t

# مشاهده logs
sudo tail -f /var/log/nginx/error.log

# Restart
sudo systemctl restart nginx
```

## 🔧 تنظیم Environment Variable برای Production

اگر API به درستی کار نمی‌کند، باید `VITE_API_URL` را تنظیم کنید:

### روش 1: در زمان Build

```bash
cd frontend
VITE_API_URL=https://taskmanager.mardomi.org/api npm run build
```

### روش 2: در Docker

در `docker-compose.production.yml`:

```yaml
frontend:
  environment:
    - VITE_API_URL=https://taskmanager.mardomi.org/api
```

### روش 3: فایل .env

در `frontend/.env.production`:

```env
VITE_API_URL=https://taskmanager.mardomi.org/api
```

## 🌐 تنظیم Nginx برای Reverse Proxy

اگر Nginx تنظیم نشده، این config رو اضافه کنید:

```nginx
server {
    listen 80;
    server_name taskmanager.mardomi.org;

    # Frontend
    location / {
        proxy_pass http://localhost:4173;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }

    # Backend API
    location /api/ {
        proxy_pass http://localhost:3000/;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

ذخیره و reload:
```bash
sudo nginx -t
sudo systemctl reload nginx
```

## 🔒 تنظیم HTTPS (SSL)

برای امنیت بیشتر، حتماً HTTPS تنظیم کنید:

### با Certbot (رایگان)

```bash
# نصب Certbot
sudo apt update
sudo apt install certbot python3-certbot-nginx

# دریافت و نصب خودکار certificate
sudo certbot --nginx -d taskmanager.mardomi.org

# تست تمدید خودکار
sudo certbot renew --dry-run
```

بعد از نصب، سایت روی `https://taskmanager.mardomi.org` در دسترس خواهد بود.

## 🐛 عیب‌یابی

### مشکل 1: سایت باز نمی‌شود

**بررسی DNS:**
```bash
ping taskmanager.mardomi.org
nslookup taskmanager.mardomi.org
```

اگر IP نمایش نداد، DNS تنظیم نشده است.

**راه حل:**
- به پنل domain بروید
- یک A Record اضافه کنید:
  - Name: `taskmanager`
  - Type: `A`
  - Value: `IP سرور شما`
  - TTL: `3600`

### مشکل 2: 502 Bad Gateway

**علت:** Backend یا Frontend در حال اجرا نیست

**راه حل:**
```bash
# چک کردن
curl http://localhost:3000
curl http://localhost:4173

# Restart
docker restart taskmanager_backend taskmanager_frontend
# یا
pm2 restart all
```

### مشکل 3: Login کار نمی‌کند

**علت:** API URL اشتباه است

**راه حل:**
1. Console browser را باز کنید (F12)
2. Network tab را ببینید
3. درخواست‌های `/api/auth/login` را چک کنید
4. اگر 404 یا CORS error دارید، `VITE_API_URL` را تنظیم کنید

### مشکل 4: CORS Error

**علت:** Backend CORS را برای domain شما allow نکرده

**راه حل:**
در `backend/src/main.ts`:

```typescript
app.enableCors({
  origin: [
    'https://taskmanager.mardomi.org',
    'http://taskmanager.mardomi.org',
  ],
  credentials: true,
});
```

سپس rebuild و restart کنید.

### مشکل 5: Mini App کار نمی‌کند

**علت:** URL در ربات تنظیم نشده

**راه حل:**
- URL ربات را به `https://taskmanager.mardomi.org` تغییر دهید
- حتماً از HTTPS استفاده کنید (HTTP کار نمی‌کند)
