کد از HamGit با موفقیت pull شد و تغییرات مهمی برای آماده‌سازی پروژه جهت استقرار (deployment) با Docker اعمال شده است.

## 📦 فایل‌های جدید (7 commit)

### 1. `.dockerignore`
فایلی که مشخص می‌کند چه فایل‌هایی نباید در Docker image کپی شوند:
- `node_modules` - پکیج‌های نصب شده
- فایل‌های `.md` - مستندات
- فایل‌های `.db` - دیتابیس محلی
- فایل `.env` - تنظیمات محیطی
- پوشه‌های `dist` - فایل‌های build شده

### 2. `Dockerfile` (برای Backend)
```dockerfile
FROM hub.megan.ir/node:18-alpine
WORKDIR /app
COPY backend/package.json backend/package-lock.json* ./
RUN npm config set registry https://hub.megan.ir/npm
RUN npm install --ignore-scripts
COPY backend/ ./
RUN npm run build
EXPOSE 3000
CMD ["node", "dist/main.js"]
```

**ویژگی‌ها:**
- استفاده از registry ایرانی `hub.megan.ir`
- نصب بدون native build با `--ignore-scripts`
- اجرای مستقیم فایل JavaScript

### 3. `Dockerfile.frontend` (برای Frontend)
```dockerfile
FROM hub.megan.ir/node:18-alpine
WORKDIR /app
COPY frontend/package.json frontend/package-lock.json* ./
RUN npm config set registry https://hub.megan.ir/npm
RUN npm install
COPY frontend/ ./
RUN npm run build
EXPOSE 4173
CMD ["npx", "vite", "preview", "--host", "0.0.0.0", "--port", "4173"]
```

## 🔧 تغییرات مهم در کد

### 1. تعویض `bcrypt` با `bcryptjs`

**قبل:**
```json
"bcrypt": "^5.1.1"  // نیاز به node-gyp و native compilation
```

**بعد:**
```json
"bcryptjs": "^2.4.3"  // Pure JavaScript، بدون نیاز به build
```

**چرا؟**
- `bcrypt` در Docker Alpine مشکل ایجاد می‌کند
- نیاز به نصب ابزارهای اضافی دارد (python, make, g++)
- `bcryptjs` کاملاً JavaScript است و مشکلی ندارد

**فایل‌های تغییر یافته:**
- `backend/package.json`
- `backend/src/auth/auth.service.ts`
- `backend/src/seed/seed.service.ts`
- `backend/src/users/users.service.ts`

### 2. تعویض `sqlite3` با `sql.js`

**قبل:**
```typescript
TypeOrmModule.forRoot({
  type: 'sqlite',
  database: 'incident_system.db',
  // ...
})
```

**بعد:**
```typescript
TypeOrmModule.forRoot({
  type: 'sqljs',
  location: 'incident_system.db',
  autoSave: true,
  // ...
})
```

**چرا؟**
- `sqlite3` یک native module است
- `sql.js` یک implementation کامل SQLite در JavaScript است
- در Docker بدون مشکل کار می‌کند

**فایل تغییر یافته:**
- `backend/src/app.module.ts`

### 3. ساده‌سازی `axios.ts`

**قبل:**
```typescript
const isTelegramWebApp = window.Telegram?.WebApp?.platform !== 'unknown';

const api = axios.create({
  baseURL: isTelegramWebApp
    ? '/api'
    : 'http://localhost:3000',
});

console.log('🌐 API Base URL:', api.defaults.baseURL);
console.log('📱 Is Telegram WebApp:', isTelegramWebApp);
```

**بعد:**
```typescript
const API_URL = import.meta.env.VITE_API_URL || '/api';

const api = axios.create({
  baseURL: API_URL,
});
```

**مزایا:**
- کد ساده‌تر و تمیزتر
- قابلیت تنظیم از طریق environment variable
- مناسب برای deployment در محیط‌های مختلف

**فایل تغییر یافته:**
- `frontend/src/api/axios.ts`

### 4. حذف محدودیت `allowedHosts`

**قبل:**
```typescript
allowedHosts: [
  'enchondromatous-aerobiologically-kristie.ngrok-free.dev',
  '.ngrok-free.dev',
  '.ngrok.io',
  'localhost'
]
```

**بعد:**
```typescript
allowedHosts: true
```

**چرا؟**
- با `true` همه domainها مجاز هستند
- مناسب برای deployment روی سرورهای مختلف
- نیازی به تغییر config برای هر domain نیست

**فایل تغییر یافته:**
- `frontend/vite.config.ts`

## 🚀 نحوه استفاده

### ساخت Docker Images

```bash
# Backend
docker build -f Dockerfile -t taskmanager-backend .

# Frontend  
docker build -f Dockerfile.frontend -t taskmanager-frontend .
```

### اجرای Containers

```bash
# Backend (روی پورت 3000)
docker run -p 3000:3000 \
  -e JWT_SECRET=your-secret \
  -e TELEGRAM_BOT_TOKEN=8787350194:AAHzqvnJhvPOh1Fg6I7L8dEbav95N6e1u04 \
  -e BALE_BOT_TOKEN=407715407:wf0tQMl7yRbnqsOhkUUyIMxb7p8simgjGmg \
  taskmanager-backend

# Frontend (روی پورت 4173)
docker run -p 4173:4173 \
  -e VITE_API_URL=/api \
  taskmanager-frontend
```

### استفاده از Docker Compose (پیشنهادی)

فایل `docker-compose.yml` بسازید:

```yaml
version: '3.8'

services:
  backend:
    build:
      context: .
      dockerfile: Dockerfile
    ports:
      - "3000:3000"
    environment:
      - JWT_SECRET=your-secret-key-here
      - TELEGRAM_BOT_TOKEN=8787350194:AAHzqvnJhvPOh1Fg6I7L8dEbav95N6e1u04
      - BALE_BOT_TOKEN=407715407:wf0tQMl7yRbnqsOhkUUyIMxb7p8simgjGmg
    volumes:
      - ./data:/app
    restart: unless-stopped

  frontend:
    build:
      context: .
      dockerfile: Dockerfile.frontend
    ports:
      - "4173:4173"
    environment:
      - VITE_API_URL=http://localhost:3000
    depends_on:
      - backend
    restart: unless-stopped
```

سپس اجرا کنید:
```bash
docker-compose up -d
```

## ⚠️ نکات مهم

### 1. حفظ دیتابیس
دیتابیس SQLite در فایل `incident_system.db` ذخیره می‌شود. برای حفظ اطلاعات:

```yaml
volumes:
  - ./data:/app
```

### 2. Environment Variables ضروری

**Backend:**
- `JWT_SECRET` - کلید رمزنگاری JWT (حتماً تصادفی و قوی باشد)
- `TELEGRAM_BOT_TOKEN` - توکن ربات تلگرام
- `BALE_BOT_TOKEN` - توکن ربات بله

**Frontend:**
- `VITE_API_URL` - آدرس API (مثلاً `/api` یا `http://api.example.com`)

### 3. امنیت در Production

❌ **نکنید:**
- فایل `.env` را commit نکنید
- توکن‌ها را در کد hardcode نکنید
- از JWT_SECRET ساده استفاده نکنید

✅ **بکنید:**
- از Docker secrets استفاده کنید
- Environment variables را از خارج تزریق کنید
- JWT_SECRET را تصادفی و حداقل 32 کاراکتر بسازید

### 4. بهینه‌سازی برای Production

```dockerfile
# Multi-stage build برای کاهش حجم
FROM hub.megan.ir/node:18-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production
COPY . .
RUN npm run build

FROM hub.megan.ir/node:18-alpine
WORKDIR /app
COPY --from=builder /app/dist ./dist
COPY --from=builder /app/node_modules ./node_modules
CMD ["node", "dist/main.js"]
```
