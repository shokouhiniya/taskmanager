# 🚀 راهنمای سریع استفاده

## شروع کار

### 1. راه‌اندازی Backend
```bash
cd backend
npm run start:dev
```
✅ باید پیام `🚀 Backend running on http://localhost:3000` را ببینید

### 2. راه‌اندازی Frontend
```bash
cd frontend
npm run dev
```
✅ باید پیام `Local: http://localhost:5173` را ببینید

### 3. Seed کردن دیتا (فقط بار اول)
```bash
curl -X POST http://localhost:3000/seed
```
✅ دسته‌بندی‌های پیش‌فرض و کاربر admin ایجاد می‌شود

## 📝 جریان کار اصلی

### برای Reporter:

#### ثبت خبر:
```
داشبورد → ثبت خبر جدید → انتخاب فرم → پر کردن فیلدها → انتخاب دسته‌بندی → ثبت
```

#### مشاهده خبرها:
```
داشبورد → خبرهای من
```

#### مشاهده خبرهای ارجاع شده:
```
داشبورد → خبرهای ارجاع شده
```

#### مشاهده اقدامات:
```
داشبورد → اقدامات من
```

---

### برای Admin:

#### ایجاد فرم:
```
داشبورد → فرم‌ساز → ایجاد فرم جدید → افزودن فیلدها → ذخیره
```

#### ویرایش فرم:
```
داشبورد → فرم‌ساز → کلیک روی ✏️ → ویرایش → ذخیره تغییرات
```

#### حذف فرم:
```
داشبورد → فرم‌ساز → کلیک روی 🗑️ → تایید
```

#### مدیریت دسته‌بندی:
```
داشبورد → دسته‌بندی‌ها → افزودن یا حذف
```

#### مدیریت خبرها:
```
داشبورد → همه خبرها → تایید / رد / ارجاع
```

#### ارجاع خبر:
```
همه خبرها → کلیک روی "📬 تایید و ارجاع" → انتخاب کاربر → ارجاع
```

#### ایجاد کاربر:
```
داشبورد → کاربران → افزودن کاربر → پر کردن فرم → ایجاد
```

#### ایجاد اقدام:
```
داشبورد → مدیریت اقدامات → ایجاد اقدام → انتخاب خبرها و کاربر → ثبت
```

## 🔧 مشکلات رایج

### خبرها نمایش داده نمی‌شوند
```bash
# 1. بررسی Console (F12)
# 2. بررسی Network (F12 → Network)
# 3. مطمئن شوید backend در حال اجرا است
# 4. مطمئن شوید token معتبر است (خروج و ورود مجدد)
```

### خطای 401 Unauthorized
```bash
# 1. خروج از سیستم
# 2. Clear Cache (Ctrl+Shift+Delete)
# 3. ورود مجدد
```

### فرم‌ها یا دسته‌بندی‌ها خالی است
```bash
# Seed کردن دیتا:
curl -X POST http://localhost:3000/seed
```

### Backend کار نمی‌کند
```bash
# 1. بررسی کنید که در پوشه backend هستید
cd backend

# 2. نصب dependencies
npm install

# 3. اجرا
npm run start:dev
```

## 🎓 نکات مهم

1. **Token Management:**
   - Token در localStorage ذخیره می‌شود
   - اگر منقضی شود، به صفحه لاگین redirect می‌شوید

2. **Access Control:**
   - صفحات محدود شده با ProtectedRoute محافظت می‌شوند
   - URL مستقیم کار نمی‌کند

3. **Data Format:**
   - data در Report به صورت JSON string ذخیره می‌شود
   - Frontend هم string و هم object را handle می‌کند

4. **Database:**
   - SQLite استفاده می‌شود
   - فایل: `backend/incident_system.db`
   - برای reset: فایل را پاک کنید و seed کنید
